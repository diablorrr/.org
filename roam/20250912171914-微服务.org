* 微服务
用于将单个应用程序按业务功能拆解为多个独立的服务，模块化（方便开发调试）、可扩展性强、容错性强（单一服务故障不会影响系统整体稳定性）

*组件* [[https://www.bagujing.com/problem-exercise/72?pid=7217][八股精]]：
- API 网关 :: 作为客户端与后端服务之间的统一入口，负责路由、负载均衡、认证和监控等
- 服务注册、服务发现 ::
  1. 服务注册：服务在启动时将自身信息注册到一个服务注册中心
  2. 服务发现：通过服务注册中心查询别的服务位置
     1) 客户端服务发现模式：客户端查询注册中心 本地执行负载均衡算法选择1个实例，客户端调用该实例
     2) 服务端服务发现模式：客户端向 负载均衡器 发送请求，负载均衡器查询注册中心 执行负载均衡算法选择1个实例，负载均衡器将请求转发到该实例
        #+begin_comment
        llfc项目中：statusserver 充当 服务注册中心、负载均衡器
        #+end_comment
- 服务间通信 :: 定义服务间通信方式（grpc）
- 负载均衡 :: 在多个服务间分配请求，实现 高可用性、高性能
- 配置管理 :: 集中管理微服务的配置，支持动态更新配置

- 消息队列 ::
- 监控和日志 :: 监控服务的运行状态，日志用于故障排除
- 容错与降级 ::
- 容器编排 ::
